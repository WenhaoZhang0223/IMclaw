# IMclaw 请求语言一致性设计

## 目标

IMclaw 的聊天回复和生成文件应使用用户当前请求的主要语言。除非用户明确要求双语，否则不得主动生成双语内容或两套不同语言的文件。

## 行为规则

- 中文请求：聊天回复、文件名、文件标题、表头、正文和说明文字均使用中文。
- 英文请求：聊天回复、文件名、文件标题、表头、正文和说明文字均使用英文。
- 混合语言请求：使用请求中占主导的自然语言；技术名词、产品名和代码标识符可以保留原文。
- 用户明确要求双语或指定另一种语言时，以用户的明确要求为准。
- 语言判断以当前请求为主，不因历史消息中使用过另一种语言而自动生成双语内容。

## 实现方案

在 `packages/imclaw/src/pi-backend.ts` 的 IMclaw 身份提示词中增加一条明确的输出语言规则。语言选择仍由模型根据当前请求判断，不增加独立的语言检测库，也不对生成结果做二次翻译或文件后处理。

该方案保持 IMclaw 对 pi-mono 的改动范围最小，不改变 Session、模型认证、文件生成或飞书文件回传流程。

## 文件回传关系

现有 `deliver_file` 行为保持不变：

1. Agent 按当前请求的主要语言生成最终文件。
2. Agent 调用 `deliver_file` 将文件发送到当前飞书私聊。
3. 只有飞书文件消息发送成功后，才删除实际回传的本地文件。
4. 上传或发送失败时保留文件，以便重试。

## 错误处理

如果当前请求语言无法明显判断，沿用用户当前消息中占比最高的自然语言。若仍无法判断，则沿用用户最近一次明确使用的自然语言。此规则只影响用户可见输出，不影响代码、路径和第三方专有名称。

## 测试

- 增加针对 IMclaw 身份提示词的单元测试，验证其明确要求匹配当前请求语言。
- 测试需覆盖：不主动生成双语、文件名与文件内容遵循同一语言、用户明确指定语言时优先服从。
- 运行 IMclaw 相关单元测试、类型检查和仓库规定的 `npm run check`。

## 非目标

- 不增加语言识别依赖。
- 不翻译用户提供的原始文件。
- 不修改 `deliver_file` 的上传、发送和删除语义。
- 不实施已暂停的 `workfile` 功能。
