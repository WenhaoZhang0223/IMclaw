# IMclaw 任务限额与临时资源清理设计

## 目标

为每个飞书任务增加强制执行的资源限制，避免 Agent 因网络采集、浏览器自动化或工具失败而长时间反复尝试。同时保证浏览器只能在后台、隔离环境中运行，并在任务结束后清理 IMclaw 创建的临时资源。

## 固定限制

每个用户任务使用独立计数和计时状态：

- 最长运行时间：10 分钟。
- 工具调用总数：最多 20 次。
- 后台浏览器启动总数：最多 2 次。

计数在每个新的用户任务开始时归零，不跨任务累计。`/status`、`/help`、`/abort` 和 `/new` 不计入 Agent 工具调用次数。

## 任务守卫

新增一个 IMclaw 层的 `TaskGuard`，由 `PiAgentBackend.prompt()` 为每次任务创建。它负责：

1. 启动 10 分钟计时器。
2. 监听 Session 的 `tool_execution_start` 事件。
3. 统计当前任务的全部工具调用。
4. 统计当前任务的浏览器启动命令。
5. 达到限制时调用当前 Session 的 `abort()`。
6. 保存明确的终止原因，供飞书回复使用。
7. 在任务退出时取消事件订阅和计时器。

达到第 20 次工具调用时仍允许该工具完成；第 21 次工具调用开始前中止任务。浏览器启动命令同理：前两次允许，第三次开始前中止。

## 浏览器安全规则

浏览器自动化采用方案 A：允许后台无界面浏览器，但禁止操作用户日常浏览器。

IMclaw 在 pi-mono 已有的 `tool_execution_start` 事件中检查 `bash` 参数。该事件会在工具参数准备和实际执行之前完成，因此守卫可以通过中止 Session 阻止违规命令，而不需要修改或替换 pi-mono 的内置 `bash` 工具。Edge、Chrome 或 Chromium 的允许启动必须同时满足：

- 包含无界面参数 `--headless` 或 `--headless=new`。
- 包含独立的 `--user-data-dir`。
- `user-data-dir` 必须位于当前 IMclaw 工作区，并使用保留前缀 `.tmp-edge-profile`。
- 不得引用用户默认浏览器配置目录。
- 不得尝试复用用户 Cookie、登录状态或已打开的浏览器调试端口。

如果浏览器命令不符合要求，守卫在命令执行前中止整个任务，并返回安全限制说明。身份提示词同时说明这些规则，减少模型反复生成被拒绝的命令。

该限制针对通过 IMclaw `bash` 工具发起的浏览器自动化。IMclaw 不提供操作用户现有浏览器或可见浏览器窗口的工具。

## 临时资源范围

以下名称被定义为 IMclaw 保留的临时资源：

```text
.tmp-edge-profile*
.tmp-cdp-*
.tmp-edge*.out
.tmp-edge*.err
.tmp-reply-urls.json
.tmp-video-body.txt
```

清理逻辑仅匹配 IMclaw 工作区根目录中的这些名称。它不得递归搜索工作区之外的位置，也不得删除：

- `deliverables`
- `workfile`
- `docs`
- 普通项目源码
- 不符合保留名称的用户文件

## 清理时机

所有任务终止路径都执行相同的 `finally` 清理：

- 正常完成。
- 10 分钟超时。
- 超过 20 次工具调用。
- 第三次浏览器启动被阻止。
- 用户发送 `/abort`。
- 模型或工具报错。

清理顺序：

1. 取消任务计时器和 Session 事件订阅。
2. 查找命令行明确引用当前工作区 `.tmp-edge-profile*` 的浏览器进程。
3. 只终止这些临时浏览器进程，不影响用户正常浏览器。
4. 删除保留名称对应的临时文件和目录。
5. 记录清理失败，但不得因为清理失败掩盖原始任务结果或终止原因。

## 飞书反馈

任务正常完成时沿用 Agent 的最终回复。

守卫主动中止时，飞书收到稳定、明确的中文说明：

- `任务已达到 10 分钟运行限制，已中止并清理临时文件。`
- `任务已达到 20 次工具调用限制，已中止并清理临时文件。`
- `任务已达到 2 次后台浏览器启动限制，已中止并清理临时文件。`
- `浏览器启动不符合后台隔离规则，任务已中止并清理临时文件。`

用户主动 `/abort` 时继续使用现有中止回复，同时执行临时资源清理。

## 组件边界

### `task-guard.ts`

负责计时、工具计数、浏览器命令校验、终止原因和生命周期释放。它不负责飞书消息发送。

### `temporary-artifacts.ts`

负责安全识别临时路径、结束只属于 IMclaw 临时配置的浏览器进程，以及删除保留的临时资源。它不得访问工作区之外的路径。

### `pi-backend.ts`

在每次 `prompt()` 中创建守卫、连接 Session 事件、执行任务并在 `finally` 中清理。它把守卫终止原因转换为稳定错误，使 Controller 能发送明确消息。

### `controller.ts`

保留现有每聊天 FIFO 队列。识别任务限制错误并直接发送其安全说明；未知错误继续使用错误编号。`/abort` 仍立即清空队列并中止 Session。

## 错误处理

- 清理某个文件失败时继续清理其余目标，并在终端记录失败路径和错误。
- 临时浏览器进程查询失败时仍尝试文件清理。
- 文件被占用时保留错误日志，不删除其他非临时文件。
- 多次触发限制时只保留第一个终止原因。
- 任务超时和用户 `/abort` 并发发生时，用户主动中止优先，不发送重复回复。

## 测试

测试使用虚假时钟、虚假 Session 和临时工作区，不启动真实浏览器或调用真实模型。

- 10 分钟前不终止，达到 10 分钟时终止。
- 前 20 次工具调用允许，第 21 次中止。
- 前两次合规后台浏览器启动允许，第三次中止。
- 缺少 `--headless` 的浏览器命令在执行前被拒绝。
- 使用工作区之外 `user-data-dir` 的命令被拒绝。
- 普通 `bash` 命令不计入浏览器启动次数。
- 正常完成、超时、限额、报错和 `/abort` 都调用清理。
- 只删除保留名称，保留 `deliverables`、`workfile` 和普通文件。
- 只结束命令行引用当前工作区临时配置的浏览器进程。
- 清理失败不覆盖原始任务错误。

完成后运行 IMclaw 相关 Vitest、类型检查和仓库规定的 `npm run check`。不调用真实飞书、浏览器或模型。

## 非目标

- 不修改 pi-mono 核心包。
- 不实现通用浏览器自动化框架。
- 不保证任意脚本内部隐藏启动浏览器时的静态识别；系统提示词禁止绕过守卫。
- 不清理 `deliverables` 或尚未实施的 `workfile` 内容。
- 不改变文件成功回传后删除生成文件的现有语义。
